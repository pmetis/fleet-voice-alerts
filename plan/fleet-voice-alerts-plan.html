Design philosophy: Each Cloud Function does one job. Firestore is the single source of truth between the add-in and backend. The add-in only writes config and reads status — it never calls Twilio or Geotab directly. ACE is called at two points: once to enrich the exception before the call, and once per supervisor question during the live call.

DAY
02
// DETECTION + ENRICHMENT

GetFeed Poller + ACE Enrichment + Call Queue

The intelligence layer. Exceptions get detected, enriched, and queued for calling.

MORNING — GetFeed Poller (Cloud Function, timer)

  • Cloud Scheduler triggers pollExceptions every 60 seconds per active database
  • Call GetFeed/ExceptionEvent using stored Geotab session + fromVersion cursor
  • Filter results against the configured ExceptionRules (by exception definition ID)
  • Update feed-cursors/{dbName} after every poll — never double-process

AFTERNOON — ACE Enrichment + Queue

  • For each matched exception, call ACE with: driver name, vehicle, exception type, last 7-day safety score, recent exception history
  • ACE returns: severity assessment, driver context summary, and opening call script (localized per config language)
  • Write call document to calls/{uuid} with status queued + ACE script
  • Respect schedule window — if outside hours, set status scheduled and skip calling

ACE Prompt Structure

  • System: "You are a fleet safety AI. Respond in {language}."
  • Context: Exception payload + driver history + safety KPIs
  • Ask: Generate a <30-second natural voice script + 5 likely supervisor questions with answers
  • Pre-cache those Q&A pairs in the call doc for fast response during live call

Session Refresh

  • Add-in re-POSTs a fresh Geotab session token to Firestore every 30 min (Geotab sessions expire)
  • Poller checks token freshness before each GetFeed call
✓ DELIVERABLE Trigger a test exception in Geotab → see a call document appear in Firestore with ACE-generated script within 90 seconds.
DAY
03
// VOICE ENGINE

Twilio Outbound Call + Bidirectional Voice Conversation

The hardest day. Get a real call happening with live Q&A. Everything else is polish.

MORNING — Outbound Call Trigger

  • initiateCall Cloud Function: watches Firestore for queued call docs, initiates Twilio outbound call to contacts[0]
  • TwiML response: <Say> the ACE opening script via Google TTS voice (WaveNet, localized language)
  • After opening, <Gather> supervisor speech for up to 10 seconds, POST to /voice-response webhook
  • Update call doc status to active when Twilio confirms call connected

AFTERNOON — Live Voice Loop

  • /voice-response Cloud Function receives transcribed speech from Twilio
  • Check pre-cached Q&A from call doc — if match found, respond instantly
  • If no cached answer: call ACE live with current exception context + supervisor question
  • ACE answer → Google TTS → base64 audio → Twilio <Play> or <Say> → loop back to <Gather>
  • Append each turn to calls/{id}/conversation[] for the status dashboard

Escalation Logic

  • Twilio no-answer / busy webhook → set status no-answer, increment contactIndex
  • If contactIndex < contacts.length: retry with next contact after 2-min delay
  • If all contacts exhausted: set status escalated, write final doc, notify via Twilio SMS fallback

Google TTS Config

  • English: en-US-Wavenet-D
  • Spanish: es-US-Wavenet-B
  • Portuguese: pt-BR-Wavenet-B
  • Speaking rate: 0.95 (clear, professional pace)
✓ DELIVERABLE Real phone gets called. Supervisor hears the ACE script. Asks a question. Gets an answer. Call ends and logs to Firestore.

Strategic Notes

// WHY CLOUD FUNCTIONS (NOT CLOUD RUN)

GetFeed polling at 60-second intervals is a perfect fit for event-driven Cloud Functions triggered by Cloud Scheduler. No idle compute cost, no containers to manage. Each function does one job and exits. Simple to deploy, simple to debug in 5 days.

// ACE IS CALLED TWICE PER EXCEPTION

  • Pre-call: Enriches context, generates opening script + cached Q&A (warm path)
  • Live call: Only invoked if supervisor asks something not in the cache (cold path, <3s budget)

// TWILIO AUDIO STRATEGY

Use Twilio's built-in <Gather speech="true"> for STT — it's battle-tested and removes the need to stream raw audio. Google STT is used via Google TTS (voice output). This avoids Media Streams complexity for a 5-day build.

// RISK REGISTER

  • Geotab session expiry: Add-in refreshes token every 30 min
  • ACE latency in live call: Pre-cached answers cover 80% of questions
  • Twilio webhook timeout (5s): Return TwiML immediately, play hold music while ACE thinks
  • Scope creep: NO new features after Day 3
